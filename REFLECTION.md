## Backend

Using AI agents during backend development helped mainly in debugging and validating complex logic. It was useful in identifying issues like inconsistent banking logic, incomplete compliance balance computation, and architectural violations, which improved the overall robustness of the system. Compared to manual coding, it reduced the time spent on debugging and reasoning through edge cases. In future, I would use AI earlier in the design phase to catch potential issues sooner and make the development process even more efficient.

## Frontend

Using AI agents during frontend development was helpful primarily for refining data flow and improving state management. It supported identifying inefficiencies in filtering logic, handling of derived state, and inconsistencies in API integration across components. This helped in structuring a cleaner separation between UI and data layers while ensuring type safety during form handling and API communication. Compared to manual development, it reduced the effort required to reason about edge cases in user input and data transformation. In future, I would involve AI earlier while designing component interactions and data flow to make the frontend more scalable and maintainable from the start.
